# ⚡ STM32 智能手持辅助系统电棒 (Smart Baton BMS)

## 📖 项目简介

这是一个基于 STM32F103C8T6 微控制器打造的极客 DIY 项目——专为手持高功率设备设计的“智能仪表盘”。它不仅能精准监测电池的充放电状态，还具备极其酷炫的高帧率粒子动画、环境温度检测以及防近身的人体雷达侦测系统。非常适合想要为自己的硬核 DIY 装备增加互动感与科技感的玩家！

## ✨ 功能亮点

* **🔋 丝滑的电池管家**
摆脱粗糙的电压跳动！系统内置了平滑算法与真实的电池曲线预估，无论是大功率放电还是插入充电器，屏幕都能精准、稳定地反馈当前的电量百分比与实时电压。
* **🎨 极客风动态交互 UI**
* **开机动画**：启动自带流动的散落粒子特效。
* **待机界面**：屏幕中拥有受物理引力影响的游动粒子，拒绝死板。
* **工作特效**：侦测到高功率放电（击发）时，立刻无缝切入专属的“闪电”高频震动特效。
* **智能充电**：插入充电器后，动态粒子自动向中心汇聚，并实时推算当前充满所需的时间（ETA）。


* **🌡️ 环境感知与雷达预警**
实时读取设备所处环境的温度；当集成的人体红外雷达检测到周围有活体靠近时，屏幕会自动浮现雷达预警“小人”图标。

## 🛒 材料清单 (BOM)

开始组装前，你需要准备以下核心硬件：

1. **主控板**：STM32F103C8T6 最小系统板（俗称 Blue Pill）。
2. **显示屏**：0.96英寸 OLED 屏幕（**注意：务必购买支持 4 线 SPI 接口的版本**，以支撑项目的高帧率动画）。
3. **温度传感器**：DS18B20 模块（引脚处需自备一颗 4.7kΩ 上拉电阻）。
4. **人体感应模块**：微型 PIR 红外雷达感应器（如 HC-SR505 或 HC-SR501）。
5. **电压采样元件**：2 颗贴片或色环电阻（推荐 36kΩ 和 10kΩ，用于将电池高压安全分压给主控芯片）。

### 1. 电池电压采样分压网络
由于 2S 锂电池组的满电工作电压高达 8.4V，远超 STM32 引脚的 3.3V 耐压极限，必须搭建高精密分压电路：
$$\\text{V\\_BATT\\_SENSE} = \\text{V\\_BATTERY} \\times \\frac{R_2}{R_1 + R_2}$$
* **推荐阻值**：$R_1 = 36\\text{k}\\Omega$, $R_2 = 10\\text{k}\\Omega$。此时理论分压比为 $(36+10)/10 = 4.6$。

本项目是一个基于 **STM32F103C8T6** 微控制器的手持自制高功率智能电棒控制与多功能显示终端。系统集成了高精度电池管理系统 (BMS)、高性能实时动态 UI 粒子渲染引擎、非阻塞环境温度状态机以及活体目标红外雷达检测。

为了解决直接从网页复制 Markdown 容易发生格式错乱、排版错位的问题，本项目特提供标准的 `README.md` 开源规范文档，支持直接导入 GitHub 仓库。

## 🚀 核心功能特性

### 1. 📊 高阶 BMS 电池监测与双级滤波算法
* **硬件级过采样**：每次采集循环连续进行 10 次高速 ADC 采样求平均值，剔除初级硬件瞬态毛刺。
* **SMA-3 & IIR 级联滤波**：代码中引入了长度为 3 的循环滑动平均滤波器 (SMA-3)，在此基础上嵌套无限脉冲响应滤波器 (IIR，权重系数 `0.7 旧值 + 0.3 新值`)。该级联架构能够完美平滑大功率击发时造成的瞬态低频电压暴跌，确保电压显示极其平稳。
* **非线性 SoC 电量精确估算**：废弃了传统的线性电压映射，建立了专为 2S 动力锂电池组设计的 `VoltageSoC_t` 查找表，支持分段线性插值算法，在电池放电平坦期（7.4V 上下）依然能提供极其精准的百分比数值。

### 2. 🎨 全动态高帧率物理交互 UI (基于 U8g2)
* **开机流体闪屏 animation**：启动时在全屏无阻塞渲染 50 个带有随机初速度矢量 (`vx`, `vy`) 的散射物理粒子，并叠加流体流动特效。
* **待机引力场粒子系统**：主界面游离 35 个动力学粒子，屏幕底部进度条被定义为“局部引力区”，当粒子下落至感应区时，系统实时计算引力加速度向量，产生受物理重力与引力吸引的炫酷视效。
* **击发放电雷电动画**：当主控侦测到电压暴跌超过放电阈值时，自动切入放电 UI，利用预设的非规则几何顶点数组和高频随机噪声（Jitter），实时绘制处于剧烈高频震动状态的“闪电”线条。
* **吸积盘充电粒子与正弦波浪**：插入充电器后，引力源自动切换至左上角，所有粒子受吸引呈现向心汇聚的吸积盘特效。同时，电池内部的填满矩阵使用 `sinf` 函数配合系统 Tick 实时生成波动的电量水平面动画。

### 3. 🌡️ 纯非阻塞外设驱动流
* **DS18B20 状态机驱动**：摒弃了传统单总线驱动中长达 750ms 的死等阻塞延时（`HAL_Delay`），将测温过程拆分为 `IDLE` (发送转换) $\rightarrow$ `START_CONVERSION` (硬件定时锁存) $\rightarrow$ `READ_DATA` (满 750ms 后读取) 的三段式状态机，利用 Cortex-M3 内核的 DWT 周期计数器实现微秒级非阻塞精确时序，确保屏幕刷新率始终保持在 50+ FPS 以上。
* **人体红外雷达防近身系统**：集成 PIR 传感器，通过硬件下拉输入检测。当持续触发超过 500ms 时，状态机翻转，并在主界面通过位置偏移算法，居中塞入高精度 16x24 像素的防近身警告“小人”图标。

### 4. ⚡ 硬件充放电锁定保护机制
* 为防止电击放电结束后，高倍率动力电池由于电化学自恢复导致电压大幅度回弹回升，代码内置了 `CHARGE_LOCKOUT_AFTER_DISCHARGE_MS` (2000ms) 的强逻辑电锁。放电停止后的 2 秒内，系统强制屏蔽充电状态机，彻底杜绝误触。
* 充电检测引入了 600ms 的硬件去抖动计时器，只有持续检测到平滑攀升的电压才会确立充电会话。

## 🛠️ 电路核心元器件选型与校准说明

### 1. 电池电压采样分压网络
由于 2S 锂电池组的满电工作电压高达 8.4V，远超 STM32 引脚的 3.3V 耐压极限，必须搭建高精密分压电路：
$$\text{V\_BATT\_SENSE} = \text{V\_BATTERY} \times \frac{R_2}{R_1 + R_2}$$
* **推荐阻值**：$R_1 = 36\text{k}\Omega$, $R_2 = 10\text{k}\Omega$。此时理论分压比为 $(36+10)/10 = 4.6$。
* **物理精度**：为了保证 BMS 的电量估算不发生严重漂移，请务必选用 **$\pm1\%$ 精度的低温漂精密贴片电阻**。
* **软件微调**：若实测电压存在系统性公差，可直接修改代码中的校准宏定义：
  ```c
  #define CAL_FACTOR 0.004007f  // 针对硬件分压比进行比例微调

```

### 2. OLED 显示屏硬件加速

* 必须购买原生支持 **4-Wire Hardware SPI** 协议的 0.96 寸 SSD1306 OLED 屏幕。
* 本项目的代码中专门针对 U8g2 进行了底层重写，彻底抛弃了低效的软件模拟 IO 时序。在 `u8x8_byte_4wire_hw_spi` 回调函数中直接调用硬件 `HAL_SPI_Transmit`，从而将屏幕刷新帧率推向极致，使物理粒子系统的动态计算毫无卡顿。

## 📂 固件主循环架构逻辑控制流

代码完全围绕高效的多任务时间片与状态机轮询展开，没有使用任何具有阻塞风险的系统死等：

```
[ 主循环 While(1) ]
       │
       ├──> 1. 高速触发 10次 ADC 采样 ──> 压入 SMA-3 环形队列 ──> IIR 深度滤波 ──> 产出稳定 smoothVoltage
       │
       ├──> 2. 状态机步进轮询：DS18B20 测温 (判断 750ms 时间戳，到期触发读取，否则立即切出)
       │
       ├──> 3. 状态机步进轮询：PIR 活体雷达检测 (高电平持续 500ms 确立目标，触发 2.5s 冷却防抖)
       │
       ├──> 4. BMS 状态判决树：
       │       ├── [A] 若电压暴跌 >= 0.25V ──> 强制切入 isDischarging=1 (放电特效最高优先级)
       │       ├── [B] 若处于放电 ──> 监测恢复 ──> 退出放电 ──> 启动 2000ms 强力充电锁定防误触
       │       └── [C] 若处于待机 ──> 监测 smoothVoltageGain >= 0.03V ──> 稳定 600ms ──> 进入充电
       │
       └──> 5. 图形渲染树：
               ├── 常驻渲染：顶部状态栏 (显示百分比图标、平滑电量、实时精确电压)
               ├── 状态分流：充电模式 (轮询渲染 5s 吸积粒子 / 3s 剩余分钟时间估算与正弦波浪)
               ├── 状态分流：日常模式 (渲染中央环境温度 / 35个基底引力场物理粒子交互)
               └── 常驻渲染：底部对称式双向收缩动画进度条

```

## 📦 编译环境与烧录测试手册

1. **集成开发环境**：推荐使用 `STM32CubeIDE`、`Keil MDK-ARM (v5.30+)` 或 `VS Code + STM32VSCodeExtension`。
2. **底层硬件依赖**：使用标准的 `STM32F1xx HAL 库`。代码中已经通过 `SystemClock_Config` 将外部 8MHz 晶振通过 PLL 锁相环倍频至 **72MHz** 最高主频。
3. **图形库安装**：在项目中引入最新版的 `U8g2` 图形库。
4. **固件部署**：
* 准备一个 `ST-Link V2`、`DAP-Link` 或 `J-Link` 仿真器。
* 将仿真器的 SWD 接口（`SWDIO`, `SWCLK`, `GND`, `3.3V`）连接至单片机核心板。
* 编译工程并生成 `.hex` 或 `.bin` 文件，利用软件将其一键刷入主控。


5. **基础功能功能点验收**：
* **雷达测试**：在 PIR 感应器前挥手，屏幕中间会平滑移动并弹出一个精致的 XBM 预警小人图标。
* **放电测试**：设备击发放电瞬间，屏幕将立刻由静态温度界面切入疯狂抖动的几何高频雷电线条。
